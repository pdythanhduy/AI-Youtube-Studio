# export/ — Final Packaged Assets

This folder contains the final production-ready output and project manifest. It is written by Stages 12 and updated throughout the pipeline.

---

## Files

### `export_manifest.json` — Pipeline state and asset registry (all stages)

**Written by:** Director AI at Stage 0. Updated after every stage.

This is the single source of truth for what the pipeline has produced and the current state of every asset.

**Top-level fields:**
```json
{
  "project_id": "YYYYMMDD_topic-slug",
  "schema_version": "1.0",
  "status": "ready_for_production",
  "pipeline_stage": "stage_12",
  "created_at": "...",
  "updated_at": "...",
  "assets": { ... }
}
```

**Asset status lifecycle:**
- `absent` → `in_progress` → `complete` → `validated`
- `needs_revision` (on critical failure)

**Do not edit this file manually** unless you are debugging a stuck pipeline and need to reset an asset status.

**Status:** Updated by Director AI after every stage.

---

### `project_report.md` — Human-readable production summary (Stage 12)

**Written by:** Director AI at Stage 12 (only when QA passes).

Contains:
- Project metadata (topic, language, style, completion date)
- Full asset table with file paths and statuses
- Script statistics (word count, scene count, visual beat count, real vs AI image split)
- QA summary (critical / content / consistency check scores)
- Editor handoff notes (what the editor needs to do next)
- Production notes (any flags, warnings, or decisions made during the run)

This is the document you hand to an editor or review yourself before uploading.

**Status after stage:** `project_report.status = complete` in manifest

---

## What Happens After Export

When `export/project_report.md` exists and `export_manifest.json` has `status: ready_for_production`:

1. Review `export/project_report.md` for any production notes or warnings
2. Review all content files using the asset table in the report
3. Generate voice audio from `voice/voice_script.txt`
4. Source real images using `visuals/image_plan.md`
5. Generate AI images using `visuals/ai_image_prompts.md`
6. Design thumbnail using `seo/thumbnail_prompt.md`
7. Edit the video using `visuals/storyboard.md` as the shot list
8. Update `voice/subtitles.srt` timecodes after audio is final
9. Upload to YouTube using `seo/seo.md` for title, description, tags

---

## What Does NOT Belong Here

The `export/` folder is for the Director AI's manifest and project report only. Do not put content files here manually — they belong in their respective subfolders (`research/`, `script/`, `visuals/`, `voice/`, `seo/`).
