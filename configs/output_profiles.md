# Output Profiles — AI YouTube Studio OS

Defines the format, encoding, and delivery specifications for every output file type. The Export Engine reads this file to ensure consistent, editor-ready delivery. Every output file must conform to its profile before the project is marked `ready_for_production`.

**Schema version:** 1.0

---

## Profile: `markdown_standard`

Applied to: `research.md`, `source_report.md`, `research_verified.md`, `story_outline.md`, `script.md`, `story_bible.md`, `storyboard.md`, `image_plan.md`, `ai_image_prompts.md`, `voice_direction.md`, `thumbnail_prompt.md`, `seo.md`

| Key | Value | Description |
|---|---|---|
| `format` | `Markdown (GitHub Flavored)` | GFM for table support |
| `encoding` | `UTF-8 (no BOM)` | Universal text encoding |
| `line_endings` | `LF` | Unix line endings |
| `max_line_length` | `none` | No hard line wrap (soft wrap in editors) |
| `heading_style` | `ATX` | Use `#` not underline-style headings |
| `table_alignment` | `left` | All table cells left-aligned |
| `link_style` | `inline` | `[text](url)` not reference-style |
| `empty_lines_between_sections` | `1` | One blank line between sections |
| `trailing_newline` | `required` | File must end with a newline character |

## Profile: `plain_text_tts`

Applied to: `voice_script.txt`

| Key | Value | Description |
|---|---|---|
| `format` | `Plain text` | No markup of any kind |
| `encoding` | `UTF-8 (no BOM)` | Required for multilingual content |
| `line_endings` | `LF` | Unix line endings |
| `max_line_length` | `100` | Soft wrap at 100 chars for readability |
| `allowed_special_syntax` | Pacing markers only | `[PAUSE:Xs]`, `[SLOW]`, `[NORMAL]`, `[FAST]`, `[WHISPER]` |
| `disallowed_characters` | `#`, `*`, `_`, `>`, `|`, `` ` `` | No markdown or code characters |
| `paragraph_separator` | `double newline` | Two line breaks between scenes |
| `trailing_newline` | `required` | |
| `bom` | `forbidden` | BOM breaks some TTS systems |

### Pacing Marker Format Specification

All pacing markers must exactly match this format (case-sensitive):

```
[PAUSE:0.5s]
[PAUSE:1s]
[PAUSE:1.5s]
[PAUSE:2s]
[PAUSE:3s]
[SLOW]
[NORMAL]
[FAST]
[WHISPER]
```

Any marker not in this list is invalid and will be flagged by the QA Engine.

## Profile: `srt_subtitles`

Applied to: `subtitles.srt`

| Key | Value | Description |
|---|---|---|
| `format` | `SubRip Text (SRT)` | Standard subtitle format |
| `encoding` | `UTF-8 with BOM` | BOM required for maximum video editor compatibility |
| `line_endings` | `CRLF` | Windows line endings — SRT standard |
| `index_start` | `1` | First subtitle index is 1 |
| `timecode_format` | `HH:MM:SS,mmm --> HH:MM:SS,mmm` | Comma as decimal separator (not period) |
| `max_chars_per_line` | `42` | Hard limit |
| `max_lines_per_segment` | `2` | Hard limit |
| `max_duration_seconds` | `7.0` | Hard limit |
| `min_duration_seconds` | `1.0` | Hard limit |
| `segment_gap_ms` | `minimum 40ms` | Gap between end of one and start of next |
| `blank_line_between_segments` | `required` | SRT standard |
| `trailing_newline` | `required` | |

### SRT Template

```
1
00:00:01,000 --> 00:00:04,500
First subtitle line here.

2
00:00:05,000 --> 00:00:08,200
Second subtitle first line
and optional second line.

```

## Profile: `audio_spec`

| Key | Value | Description |
|---|---|---|
| `format` | `MP3` or `WAV` | WAV for masters, MP3 for delivery |

## Profile Selection Rules

The Export Engine selects the profile for each file automatically based on file extension and canonical name:

| File Pattern | Profile Applied |
|---|---|
| `*.md` (all project output) | `markdown_standard` |
| `voice_script.txt` | `plain_text_tts` |
| `subtitles.srt` | `srt_subtitles` |
| `*.json` | `json_manifest` |
| `assets/music/*.mp3`, `*.wav` | `audio_spec` |

## Validation Against Profile

The QA Engine and Export Engine both validate files against their profiles. A file that violates its profile is flagged with severity:

| Violation Type | Severity |
|---|---|
| Wrong encoding | HIGH — may cause display errors |
| BOM present in plain text | MEDIUM — breaks some TTS systems |
| SRT timecode format wrong | HIGH — subtitles will not import |
| Max line length exceeded (SRT) | HIGH — display overflow |
| Max duration exceeded (SRT) | MEDIUM — readability issue |
| Trailing comma in JSON | HIGH — JSON parse failure |
| Pacing marker syntax wrong | MEDIUM — TTS will not interpret |
| Markdown in plain text | HIGH — TTS will read markup aloud |
